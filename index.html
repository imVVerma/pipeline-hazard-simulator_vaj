About This Simulator

This Pipeline Hazard Simulator helps visualize how instructions move through a CPU pipeline cycle-by-cycle, making it easier to understand RAW hazards, stalls, and data forwarding in computer architecture.

Built by three students as part of the Fundamentals of Computer Science (FoCS) course.

How to Use

  • Enter instructions in MIPS assembly format (e.g., add $t0, $t1, $t2, lw $t0, 0($t1)).
  • Choose a 4-stage (IF, ID, EX, MEM/WB) or 5-stage (IF, ID, EX, MEM, WB) pipeline.
  • Enable/disable data forwarding to compare execution behavior.
  • Run the simulation to view instruction flow, stalls, and hazard resolution cycle-by-cycle.
